# 合同审核规则手册

## 1. 目的

本手册用于指导 Agent 审核 ABB 机器人裸机销售相关文件，并在审核通过后生成 BT09 开启邮件。
**适用文件：**
- 销售合同
- CQP
- TA（技术协议）
- 其他相关附件（如诚信协议、付款附件等）

## 2. Agent 的核心任务

**Agent 必须完成以下任务：**
- 识别并读取合同、CQP、TA
- 核对三者之间的关键信息一致性
- 判断 Incoterm（DDP / EXW）
- 按规则检查配置、金额、税率、交货、付款条件等
- 区分 BLOCKER 与非 BLOCKER
- 审核通过后，生成 BT09 邮件草稿

## 3. 审核总流程

### Step 1：确认文件是否齐全

**至少应有：**
- 合同
- CQP
- TA

#### TA 文件识别补充 在部分电子签场景下，TA 不会单独作为文件上传，而是直接附在合同文件中。

**规则：**
Agent 必须先检查合同附件中是否已包含 TA 内容
- 若合同中已包含 TA，则视为 TA 文件已提供
- 仅在既无单独 TA 文件、合同中也无 TA 内容时，才判定 TA 缺失
- 用户至少应上传合同+对应的CQP 两个文件

### Step 2：提取基本信息

**Agent 应提取并核对以下字段：**
- 合同号
- CQP号 / 版本号
- 买方名称
- 买方地址
- 最终用户名称
- 最终用户地址
- 机器人型号
- 数量
- 销售人员
- PM（如文件中可获取）
- 质保信息
- 交货条款
- 付款条件
- 含税/未税金额
- 增值税税率

### Step 3：先判断 Incoterm（DDP / EXW）

Agent 必须先判断是 DDP 还是 EXW，再决定使用哪个 BT09 模板。

### 3.1 从合同判断 DDP

若合同中勾选了以下描述，且“交付地点”不为空白，则判定为 DDP：

> 卖方工厂的到货价。设备从上海自由贸易试验区发出，当设备到达指定交付地点时，设备的风险即从卖方转移至买方。
**判断条件：**
- 勾选“卖方工厂的到货价”
- 交付地点不为空白
**则：**
- Incoterm = DDP
- BT09 邮件中使用 DDP 模板

### 3.2 从合同判断 EXW

若合同中勾选了以下描述，且“交付地点”为空白，则判定为 EXW：

> 买方工厂出厂价。当卖方通知买方设备已在上海自由贸易试验区工厂内包装完毕可以发运时，设备的风险即从卖方转移至买方。
**判断条件：**
- 勾选“买方工厂出厂价”
- 交付地点为空白
**则：**
- Incoterm = EXW
- BT09 邮件中使用 EXW 模板

### 3.3 用 CQP 进行二次验证

CQP 中通常可看到类似字段：

- 交货条款
- DDP Shanghai 以 INCOTERMS 2010 为准（特殊指定的单项产品标准需另予注明）

**或：**

- EXW Shanghai 以 INCOTERMS 2010 为准（特殊指定的单项产品标准需另予注明）
**规则：**
- 合同与 CQP 中的 Incoterm 必须一致
- 若不一致，必须标注为问题
- 若无法明确判断 Incoterm，不能直接放行

### 3.4 Incoterm 相关逻辑

DDP：通常有明确交付地点，卖方负责运输到指定地点
EXW：通常无交付地点，买方自行安排提货

## 4. 基本信息审核规则

### 4.1 合同号

**需核对：**
- 合同中的合同号
- CQP 中关联合同号 / 项目号（如有）
- TA 中项目关联信息（如有）
**规则：**
- 合同号应清晰、唯一
- 合同号如出现不一致，需标注

### 4.2 买方 / 最终用户信息

**需核对：**
- 买方名称
- 买方地址
- 最终用户名称
- 最终用户地址
**规则：**
- 名称需一致
- 地址需完整
- DDP/EXW 场景下，Ship to 逻辑需正确

### 4.3 机器人型号与数量

**需核对：**
- 合同中的型号、数量
- CQP 中的型号、数量
- TA 中的型号、数量
**规则：**
- 型号必须一致
- 数量必须一致
- 多型号项目需逐项核对

## 5. 配置审核规则

### 5.1 配置审核原则

Agent 必须对 CQP 与 TA 的配置进行逐项比对，并列出所有不一致项。
**规则：**
- 所有配置不一致项都要标注出来
- 但若中文 / 英文 / 代码中任意一种可以明确对应上，则可视为一致
- 不能因为 TA 使用中文、英文描述而直接判为不一致

### 5.2 配置一致的判定标准

**以下任意一种对上，都可视为配置一致：**
- 配置代码一致
- 英文描述一致
- 中文描述一致
- 中文 / 英文 / 代码三者可明确映射到同一配置项
**例如：**
- 3016-2
- FlexPendant with 10m cable
- 示教器10米电缆
三者若可映射到同一配置，则视为一致。

### 5.3 配置输出要求

**Agent 输出时必须：**
- 列出已匹配项
- 列出未匹配项
- 列出疑似翻译差异但本质一致的项
- 清楚说明“不一致”的具体字段，不可笼统写“配置不一致”

### 5.4 商务类代码

部分代码可能属于商务/打包/交付属性，不一定要求在 TA 中完整体现。
**例如：**
- Warranty
- Delivery project
- Bundle
- Program package（视具体情况）
**规则：**
Agent 应优先区分“技术配置项”与“商务/打包项”
- 技术配置不一致应重点提示
- 商务类项如仅表达方式不同，应说明，不要误判

## 6. 付款条件审核规则

### 6.1 唯一来源

- BT09 邮件中的付款条件，必须直接复制合同附件二中的付款条件原文。
**不得：**
- 自行总结
- 自行改写
- 自行压缩成“10/40/50”或“20/80”一句话替代
**必须：**
- 保留原文段落
- 保留序号
- 保留标点
- 保留“注”内容（如有）

### 6.2 示例

例如合同附件二写的是：

```text
付款条件：
裸机合同付款条件如下：
1） 合同总价的百分之十，应在签订本合同后七个工作日内以电汇方式支付给卖方；
2） 合同总价的百分之四十，应在设备发货前七个工作日内以电汇方式支付给卖方。
3） 合同总价的百分之五十，应在设备发货前七个工作日内以ABB 认可的电子银行承兑汇票方式支付给卖方。卖方收到全款且发货后向买方开具全额增值税发票。
注：如分批发货，除预付款外，其他款项应按批次支付。增值税发票可在收到每批设备全款后按批次开具。
```

则 BT09 邮件中必须原样引用，不可改写。

## 7. 金额与税率审核规则

### 7.1 增值税税率

**规则：**
- 税率必须明确填写
- 中国区标准裸机合同通常应为 13%
- 未填写或明显错误必须标出

### 7.2 金额核对

**需核对：**
- 合同总价（未税）
- 合同总价（含税）
- CQP 中金额
**规则：**
- 需检查合同与 CQP 金额是否基本一致
- 若有四舍五入导致的小额差异，可注明
- 金额异常需列出具体差额

## 8. 交货时间审核规则

### 8.1 核对逻辑

Agent 应同时读取：
- 合同中的交货时间
- CQP 中的交货时间

### 8.2 输出逻辑

- 若两者一致，直接通过
- 若两者不一致，必须标注
- 最终 BT09 邮件中的交货时间，以合同为准，除非用户明确另行说明

## 9. 客户 ID / GIS 号 / 地址库规则

后续将有常见客户地址、客户 ID、GIS 号的 xlsx 文件作为知识库上传。
**规则：**
Agent 应优先从知识库 xlsx 中匹配客户 ID、End customer ID、GIS号
- 若匹配成功，应自动填入 BT09 邮件
- 若匹配失败，再提示人工补充
- 因此在成熟版本中，理论上需要人工手填的部分仅保留：
- GM
- NM

## 10. BLOCKER 与非 BLOCKER

### 10.1 BLOCKER

**以下情况应优先标注为 BLOCKER：**
- 缺少合同 / CQP / TA 关键文件
- 合同与 CQP 的 Incoterm 不一致
- 无法判断 DDP / EXW
- 机器人型号不一致
- 数量不一致
- 关键技术配置无法对应
- 增值税税率缺失或明显错误
- 付款条件缺失
- 合同附件二付款条件无法提取
- BT09 关键字段无法确认且影响流程开启

### 10.2 非 BLOCKER

**以下情况通常不应直接阻断，但需提示：**
- CQP 与合同交货时间不同，但合同可作为最终依据
- 中英文表述不同，但经映射后实为同一配置
- 少量金额四舍五入差异
- 客户 ID / GIS 号待知识库补充

## 11. BT09 邮件生成规则

### 11.1 模板选择优先级

**Agent 必须先判断：**
- DDP 或 EXW
- 是否分批发货
- 付款条件原文
- 是否能从知识库补齐客户 ID / GIS号

### 11.2 输出要求

**BT09 邮件应：**
- 使用对应模板
- 付款条件直接复制合同附件二
- 客户 ID / GIS号优先从知识库提取
- 若无法提取，仅保留空缺项
- 在成熟版本中，原则上只保留 GM / NM 手填

## 12. Agent 输出格式

**Agent 输出建议采用以下结构：**
- 最终结论
- 通过 / 通过但有备注 / Blocked
- Incoterm 判断
- DDP / EXW
- 判断依据（合同 + CQP）
- 核对通过项
- 合同号
- 买方
- 最终用户
- 型号
- 数量
- 税率
- 金额
- 交期
- 付款条件
- 配置一致性
- BLOCKER 问题
- 非 BLOCKER 备注
- BT09 邮件草稿（如可生成）

## 13. 总原则

- 先判断 DDP / EXW，再选模板
- 合同与 CQP 的 Incoterm 必须一致
- 配置不一致项全部列出
- 中文 / 英文 / 配置代码任一对应即可算一致
- 付款条件必须直接复制合同附件二原文
- 客户 ID / GIS号优先从知识库 xlsx 自动匹配
- 成熟版本中理论上只保留 GM / NM 人工填写

#### 质保条款检查

Agent 必须检查合同 5.2 的质保条款。
**规则：**
- 18/12 视为 Standard Warranty
- 超过 18/12 视为 Extended Warranty
随后需核对 CQP 与 TA 中关于 Warranty 的配置或描述是否一致。 若合同、CQP、TA 对质保类型判断不一致，必须标注出来。
