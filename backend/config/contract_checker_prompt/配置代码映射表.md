# 配置代码映射表

## 1. 目的

本文件用于指导 Agent 对 CQP 与 TA 的配置进行映射和一致性判断。
**重要原则：**
- 配置不一致项必须全部列出
- 但若中文 / 英文 / 代码中任意一种可对应到同一配置项，则可视为一致
- Agent 不应因为语言不同而误判为不一致

## 2. 配置一致性判定原则

**以下任一情况满足，即可判定为“配置一致”：**
- 配置代码一致
- 英文描述一致
- 中文描述一致
- 英文描述、中文描述、配置代码可映射到同一配置项

## 3. 标准化比对方法

**Agent 在比对前，应先做标准化：**

### 3.1 文本标准化

- 忽略大小写差异
- 忽略多余空格
- 忽略中英文括号差异
- 忽略常见连接符差异（如 -、/）
- 允许中文与英文互相映射后再判断

### 3.2 术语归一化

**例如：**
- FlexPendant = 示教器
- Manipulator = 机器人本体 / 机械臂
- Controller = 控制柜 / 控制器
- Operating temperature = 工作温度
- Mains voltage = 电源电压
- Cable gland = 电缆密封接头
- Collision detection = 碰撞检测
- Path Recovery = 路径恢复
- Multitasking = 多任务
- SafeMove Standard = SafeMove标准版

## 5. 比对逻辑

### 5.1 Agent 必须做的事

**Agent 对 CQP 与 TA 配置比对时，必须：**
- 先抽取 CQP 中的配置代码/描述
- 再抽取 TA 中的配置代码/英文描述/中文描述
- 将 TA 中的中文、英文表达统一映射
- 再做一致性判断
**输出：**
- 一致项
- 不一致项
- 疑似翻译差异项

### 5.2 输出要求

**如果发现不一致，不能只写“配置不一致”，必须写成：**
- CQP：3016-3 / FlexPendant 30m
- TA：3016-2 / FlexPendant 10m
- 结论：示教器电缆长度不一致
**或：**
- CQP：3200-3 / Manipulator cable 15m
- TA：本体电缆7米
- 结论：本体电缆长度不一致

## 8. 商务 / 非技术项提醒

某些项目可能属于商务或打包类表达，Agent 应谨慎判断，不宜直接按技术不一致处理。
**例如可能包括：**
- Warranty
- Delivery project
- Bundle
- Program feature bundle
**规则：**
- 若仅为商务类表达差异，应提示但避免误判
- 若为技术参数差异，必须明确标出
