===== # BT09邮件模板库.md =====
BT09邮件模板库
1. 使用原则
Agent 生成 BT09 邮件时，必须遵循以下原则：
先判断 DDP / EXW
合同与 CQP 的 Incoterm 必须一致
根据 DDP / EXW 选择模板
付款条件必须直接复制合同附件二原文
客户 ID / GIS 号优先从知识库 xlsx 自动获取
成熟版本中原则上只保留 GM / NM 需要人工填写
2. 模板选择逻辑
模板优先级说明
如知识库中上传了《开启流程 模板.docx》，Agent 生成 BT09 邮件时应优先参考该模板的版式、字段顺序、措辞和格式。
规则：
模板控制输出格式
合同 / CQP / TA 控制实际内容
若模板与业务事实冲突，以合同 / CQP / TA 为准
2.1 先判断 DDP / EXW
选择 DDP 模板
满足以下条件时，使用 DDP 模板：
合同勾选“卖方工厂的到货价”
且交付地点不为空白
且 CQP 中交货条款为 DDP Shanghai（或同类 DDP 表述）
选择 EXW 模板
满足以下条件时，使用 EXW 模板：
合同勾选“买方工厂出厂价”
且交付地点为空白
且 CQP 中交货条款为 EXW Shanghai（或同类 EXW 表述）
2.2 再判断是否分批发货
若合同写明分批交货，则在交货时间处写明分批信息
若合同为一次性交货，则按一次性交货输出
3. 通用输出规则
3.1 邮件中必须包含
买方名称
数量
机器人型号
合同号
PM
销售人员
CQP号
交货时间（以合同为准）
付款条件（合同附件二原文）
Ship to信息
Freight term / Terms of delivery description
End customer
客户ID / GIS号（优先自动查）
GM / NM
3.2 付款条件处理规则
付款条件部分必须：
直接复制合同附件二原文
不可改写
不可概括成“20/80”“10/40/50”
如有“注”，必须保留
3.3 客户信息处理规则
Ship to ID：优先从 xlsx 知识库查找
End customer ID：优先从 xlsx 知识库查找
GIS号：优先从 xlsx 知识库查找
若知识库未命中，再保留空白待人工补充
成熟版本中理论上应仅保留：
GM
NM

EXW Ship-to补充规则：
EXW场景下，合同“交付地点”为空白通常是正常的，因为属于客户自提
但BT09中的Ship to信息不能留空
若合同未单独写明其他收货方，EXW的Ship to名称和地址默认取合同首页买方名称和买方地址
“客户自提”可以作为说明性文字保留，但不能替代公司名称和地址
不要将最终用户地址或设备安装地点直接作为EXW的Ship to地址，除非合同明确写明
6. 分批发货模板补充规则
如果为分批发货，交货时间处建议写为 交货时间： 第一批：{第一批交期与数量} 第二批：{第二批交期与数量} 第三批：{第三批交期与数量} ...
付款条件部分仍然： 优先直接复制合同附件二原文 若附件二原文中已写“注：如分批发货……”，必须完整保留
7. 示例：20/80 付款条件原文引用
若合同附件二原文为 20/80，则同样必须整段复制，不可只写“20/80”。
8. 空白字段规则
在知识库未覆盖前，允许保留空白： Ship to ID End customer ID GIS号 GM NM 在客户地址 / ID / GIS xlsx 上传后，原则上应自动填充前三项，仅保留： GM NM
9. 输出时的注意事项
不要自行润色付款条件 不要把 DDP / EXW 用错 不要把合同交期替换成 CQP 交期 不要漏掉“注”内容
