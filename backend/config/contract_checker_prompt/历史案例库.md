# 历史案例库

## 1. 用途

本案例库用于帮助 Agent 理解 ABB 合同审核中的常见场景、特殊场景与判断逻辑。
Agent 在审核时，应优先遵循《合同审核规则手册》，并结合本案例库理解实际业务判断方式。

## 2. 案例结构说明

**每个案例建议包含：**
- 合同号
- 买方
- 最终用户
- 机器人型号与数量
- Incoterm
- 交货时间
- 付款条件
- 关键审核点
- 最终处理结论
- 经验总结

## 3. 案例一：DDP 项目，付款条件为 10/40/50

#### 基本信息

- 合同号：M4367-3297
- 买方：上海华太机器人工程有限公司
- 最终用户：苏州光韵达自动化设备有限公司
- 机器人：5台 IRB 2600-12/1.85
- Incoterm：DDP Shanghai
- 付款条件：10% + 40% + 50%（以附件二原文引用）

#### 关键审核点

- 合同中勾选“卖方工厂的到货价”
- 交付地点不为空白
- CQP 中交货条款为 DDP Shanghai
因此判定为 DDP
- BT09 邮件需使用 DDP 模板
付款条件不能简写为“10/40/50”，必须复制附件二原文

#### 特殊情况

- 合同交期与 CQP 交期存在差异时，最终以合同为准
Ship to 与 End customer 地址可能不同

#### 经验总结

- 先判断 DDP / EXW，再决定模板
DDP 需要特别注意交付地点不为空
- 付款条件必须完整复制附件二，不可概括

## 4. 案例二：EXW 项目，买方工厂出厂价

#### 基本信息

- 场景：标准 EXW 裸机合同
- 合同中勾选“买方工厂出厂价”
- 交付地点为空白
- CQP 中显示 EXW Shanghai

#### 关键审核点

- 合同为 EXW
- CQP 也为 EXW
- 合同与 CQP 一致
- BT09 邮件使用 EXW 模板
Ship to 一般使用买方信息

#### 经验总结

EXW 判断不能只看 CQP，也要看合同中的勾选项与交付地点是否为空
- 合同与 CQP 的 Incoterm 必须一致

## 5. 案例三：配置项语言不同，但本质一致

#### 场景

- CQP 中使用代码与英文描述，TA 中使用中文描述。
**例如：**
- CQP：3016-2 / FlexPendant 10m
- TA：示教器10米电缆

#### 判断

虽然表达语言不同，但本质对应同一配置项
- 应判定为一致
- 不应误报为配置不一致

#### 经验总结

配置判断不能只做字面比较
- 中文 / 英文 / 代码任一可映射，即可视为一致

## 6. 案例四：配置项真实不一致

#### 场景

- CQP：3016-3 / FlexPendant 30m
- TA：3016-2 / FlexPendant 10m

#### 判断

这是示教器电缆长度真实不一致
- 必须标出具体差异
不能笼统写“配置不一致”

#### 推荐输出方式

- CQP：3016-3 / FlexPendant 30m
- TA：3016-2 / FlexPendant 10m
- 结论：示教器电缆长度不一致

#### 经验总结

- 所有配置不一致项都要逐项列出
配置不一致需要说明差异点到底是什么

## 7. 案例五：付款条件必须引用附件二原文

#### 场景

- 合同附件二中为 10/40/50，且包含分批发货注释。

#### 正确做法

BT09 邮件中直接复制：

```text
付款条件：
裸机合同付款条件如下：
1）……
2）……
3）……
注：……
```

#### 错误做法

- 只写“10/40/50”
- 自行改写成简版
- 漏掉“注：如分批发货……”

#### 经验总结

- 付款条件必须以附件二为唯一来源
原文复制比“总结”更安全

## 8. 案例六：合同与 CQP 的 Incoterm 不一致

#### 场景

- 合同判断为 DDP
- CQP 写的是 EXW Shanghai

#### 判断

Incoterm 不一致
必须标注为问题
- 不应直接生成通过结论

#### 经验总结

Incoterm 是模板选择的前提
- 合同与 CQP 不一致时不能忽略

## 9. 案例七：客户 ID / GIS 号后续从知识库自动提取

#### 场景

当前项目的客户名称、地址已在常见客户 xlsx 中维护。

#### 正确做法

Agent：
读取买方名称与地址
在 xlsx 中匹配客户 ID / End customer ID / GIS号
自动写入 BT09 邮件

#### 若未命中

提示“知识库未命中，需要人工补充”
- 不应随意编造 ID 或 GIS号

#### 经验总结

- 知识库命中则自动填
- 未命中再人工补充
- 成熟版本中原则上仅 GM / NM 保留人工填写

## 10. 案例八：交货时间不一致的处理

#### 场景

- 合同：合同生效且收到预付款后9周
- CQP：自合同生效之日起8周

#### 处理逻辑

Agent 必须标注“合同与 CQP 交期不一致”
但在最终 BT09 邮件中，交货时间以合同为准
- 若用户另有明确说明，则按用户说明执行

#### 经验总结

- 交期冲突要提示
BT09 输出默认以合同为准
